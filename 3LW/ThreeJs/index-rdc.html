Three.js - RdC Case Study

Introduzione

Questo file è il file esatto utilizzato nel sito di Roberta di Camerino (link).

Il file in questo momento è settato per il caricamento di un modello collada (.dae) generato con Cinema4D (riga 36). Nello stato attuale non necessita della presenza di una camera o di un animazione. E' stato applicato un leggero movimento collegato al mouse un una camera con Orbit Control. Il materiale viene caricato dalla stessa cartella ed è stato applicato il canale alpha. Il "sistema" non riconosce autonomamente se un oggetto ha bisogno o no di una texture alpha. E' per questo che bisogna inserire il nome dell'oggetto da escludere (come ad esempio la "Light") nella lista dei "forbidden names" (riga 508). Inserendo un elemento nella lista infatti, sarà escluso dalla ricerca dellla texture alpha corrispondente, la cui path viene definita alla (riga 556).

Testato con export da Cinema r17 - Collada versione 1.4

  • Rispetto ad un normale progetto Three.js ha bisogno di questi file aggiuntivi: ColladaLoader, Detector, inertiaScroll
  • Generare modello in Cinema 4D
  • Generare texture con Body Paint, applicare la texture come jpg e tenersi il sorgente in PSD
  • Generare l'eventuale alpha dalla stessa sorgente PSD
  • Caricare il file nella riga 36 (settare bene la path dal file che carica il JS)
  • Inserire gli elementi che non hanno il canale alpha nei "forbidden names", alla riga 508
  • Settare la path del canale alpha (se necessario), alla riga 556
